Mastering Kubernetes: A Deep Dive into Pods

TW
Tri Wantoro Author
Oct 24, 2025 5 min read

Kubernetes has become the de facto standard for container orchestration. But at its core, it relies on a simple yet powerful concept: the Pod.

What is a Pod?

A Pod is the smallest deployable unit of computing that you can create and manage in Kubernetes. A Pod (as in a pod of whales or pea pod) is a group of one or more containers, with shared storage and network resources, and a specification for how to run the containers.

apiVersion: v1
kind: Pod
metadata:
  name: nginx
spec:
  containers:
  - name: nginx
    image: nginx:1.14.2
    ports:
    - containerPort: 80

Why not just run containers directly?

While you could run containers directly (like in Docker), Pods provide a higher level of abstraction that allows for co-located processes (like a sidecar container for logging) to share the same IP address and port space.
